## 1. Root-Cause Diagnostic Report

### 1.1 Scalper Archetype — True Root Cause (Not Label Inversion)

| Model | OOS Acc | Baseline (random 3-class) | Diagnosis |
|---|---|---|---|
| CNN_Scalper_v1 | 0.1056 | 0.333 | **CRITICAL** |
| GRU_Scalper_v1 | 0.1417 | 0.333 | **CRITICAL** |
| LinearAttn_Scalper_v1 | 0.4719 | 0.333 | DEGRADED |

**Confirmed diagnosis:** Missing `StandardScaler` in `evaluate_all_checkpoints.py → load_scalper_data()`.  
The model was trained on features normalised with `FeatureFactory.fit_scaler_train_only()` + `transform_with_scaler()`.  
The eval script calls `_build_scalper_features()` but **never applies the scaler**, so raw feature values go into a model expecting normalised inputs. This causes the model to collapse to a single-class prediction (≈10% accuracy) — *not* a label-inversion bug.

**Label convention is CORRECT and consistent:**
- `0` = down (`fwd_ret < -0.0003`)
- `1` = flat (|fwd_ret| ≤ 0.0003)  
- `2` = up (`fwd_ret > 0.0003`)

Same in `scalper_data.py` and `evaluate_all_checkpoints.py`. No inversion exists.

**Fixes applied:**
1. `quant_core/scalper_data.py` — save fitted scaler to `{ckpt_dir}/feature_scaler.pkl` alongside `model_best.pt`
2. `evaluate_all_checkpoints.py` — load `feature_scaler.pkl` and apply it before building sequences
3. `quant_core/scalper_training.py` — save per-epoch checkpoints `epoch_{n}.pt` after each epoch

### 1.2 Discretionary Archetype — Renderer Mismatch (Confirmed)

The evaluation renderer in `load_disc_data()` was **completely different** from the training renderer:

| | Channel 0 | Channel 1 | Channel 2 | Channel 3 |
|---|---|---|---|---|
| **Training** (`discretionary_data.py`) | open_norm (per-bar) | high_norm | low_norm | close_norm |
| **Eval** (`evaluate_all_checkpoints.py`) | close price line | volume bars | (empty) | log-return sparkline |

This is a fundamental image representation mismatch. Models trained on 4-channel OHLCV pixel images are given completely different images during eval — their weights are useless for this input format. The **checkpoints are NOT garbage**; they trained successfully, they were just evaluated incorrectly.

**Fix applied:** `evaluate_all_checkpoints.py → load_disc_data()` now imports and calls `_rasterize_window()` directly from `quant_core.discretionary_data` to guarantee identical encoding.

## 3. Orchestrator Architecture (Phase B)

### 3.1 Queue Design

```
BATCH_1: StatArb (GAT, Autoencoder, LSTM)          ← runs first; uses existing ✅ GAT checkpoint
BATCH_2: Scalper (CNN, LinearAttn, GRU)             ← runs after Phase A renderer/scaler fix
BATCH_3: Market Maker (PPO from scratch, SAC, DQN)  ← PPO has no checkpoint; full retrain
BATCH_4: Trend residuals (Transformer, LSTM)        ← improve Sharpe 0.38/1.40 → target > 1.5
BATCH_5: Mean Reversion residuals (ResNet, GRN)     ← Sharpe 0.61/-1.1 need full retrain
BATCH_6: Discretionary (ViT, Multimodal, CNNChart)  ← requires renderer fix in Phase A
```

### 3.2 Fault Tolerance

- Each model training is wrapped in `try/except`
- Failures are written to `error_log.json` (append-safe)
- Execution continues to the next model without hanging
- `last_completed_checkpoint.json` per model dir enables resume from last completed epoch

### 3.3 Per-Epoch Checkpointing Strategy

```
models/checkpoints/{archetype}/{model_name}/
├── model_best.pt           ← best validation checkpoint (existing)
├── optimizer_state.pt      ← optimizer state for resume (existing)
├── epoch_1.pt              ← NEW: per-epoch weights
├── epoch_2.pt
├── ...
├── epoch_N.pt
├── feature_scaler.pkl      ← NEW: fitted StandardScaler (scalper only)
└── last_completed_checkpoint.json   ← NEW: {"epoch": N, "val_loss": 0.xx, "ts": "..."}
```

**Retention policy:** Keep last 3 epochs only (auto-delete older) to prevent disk exhaustion.

### 3.4 Hardware

All tensor operations forced to **AMD Radeon RX 6750 via DirectML** (`preferred_backend: directml`).  
CPU fallback applies only if DirectML import fails.
